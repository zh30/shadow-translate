### 1. 产品概述

本产品是一款**完全本地化（on-device）**运行于 macOS 的 AI 翻译客户端 App，定位为 DeepL Mac 客户端的本地化、高隐私替代方案。核心差异在于：

- 采用 Hugging Face mlx-community/translategemma-4b-it-4bit_immersive-translate（MLX 4-bit 量化版本，约 2.18 GB 模型文件，基于 Google TranslateGemma-4B-IT 优化），利用 Apple Silicon 的统一内存和 Metal 加速，实现**零网络依赖**、**实时翻译**与**数据绝对隐私**。
- 聚焦“鼠标选中即译”的极致快捷体验，同时提供永久本地历史记录用于搜索、分析与复用。

**产品定位**：面向需要高频、隐私敏感翻译的用户（程序员、研究者、跨境工作者、多语种内容创作者），在 Mac 生态中提供比云端翻译工具更快的响应、更强的本地控制，以及 DeepL 无法实现的“完全离线 + 可替换原文本”能力。

**核心价值主张**：

- **隐私至上**：所有翻译在本地完成，无数据上传、无 API 调用。
- **性能优化**：MLX 框架专为 Apple Silicon 设计，在 M 系列芯片上实现秒级甚至亚秒级翻译（视文本长度与硬件而定）。
- **无缝集成**：全局快捷键 + 智能弹窗 + 上下文替换，极大提升日常翻译效率。
- **持久价值**：翻译历史永久本地存储，支持搜索、标签、导出与简单分析。

### 2. 产品目标与成功指标

**业务目标**：

1. 提供媲美 DeepL 弹窗体验的本地翻译工具，降低用户对云服务的依赖。
2. 利用 MLX 模型为 Mac 用户带来更强翻译能力（支持广泛语种，专注沉浸式/精确翻译）。
3. 构建用户个人“翻译知识库”，实现历史复用与长期价值。

**用户目标**：

- 1 秒内完成选中文本翻译并复制/替换。
- 历史记录可搜索、可分析，避免重复劳动。
- 完全离线使用，无隐私担忧。

**成功指标（KPI）**（MVP 后 3 个月）：

- 日活跃用户（DAU）转化率 > 40%（安装后首周）。
- 平均每日翻译次数 ≥ 20 次/用户。
- 历史记录使用率 ≥ 30%（用户查看/搜索历史）。
- 用户满意度（NPS）≥ 70。
- 崩溃率 < 0.5%，翻译延迟 < 2 秒（95% 分位）。

### 3. 目标用户与用户画像

**主要用户**：

- **技术开发者/工程师**（Mac 重度用户）：代码注释、文档、StackOverflow 多语种阅读。
- **研究者/学者**：学术论文、跨语言文献翻译与存档。
- **内容创作者/翻译爱好者**：博客、社交媒体、多语种写作。
- **隐私敏感专业人士**：律师、医生、金融从业者（处理敏感文档）。

**次要用户**：学生、跨境电商从业者。

**痛点**：

- 云端翻译隐私风险、延迟、配额限制。
- 现有工具无法在输入框中“一键替换”。
- 翻译历史临时或云端存储，无法长期本地搜索分析。

### 4. 功能需求

#### 4.1 核心功能模块

**模块 1：翻译引擎集成**

- 集成指定模型：`mlx-community/translategemma-4b-it-4bit_immersive-translate`（MLX 版本）。
- 使用 MLX 框架（支持 Python/Swift 绑定）进行本地推理，支持流式输出（streaming）以提升感知速度。
- 支持输入格式：
  - 字符串占位符：`<<<source>>>en<<<target>>>zh<<<text>>>Hello world`
  - 结构化消息（兼容 OpenAI 格式）。
- **自动检测（Auto）**：App 层面实现（先用系统语言检测 API 或轻量本地检测器预判；模型不支持原生 auto 时，App 可尝试 top-N 语言并选最佳）。
- 支持语种：模型原生能力（英文、中文、法文、德文、西班牙文等常见语种；详细列表以模型 Card 为准，后续可扩展 fine-tune）。
- 高级设置：温度（temperature）、top-p 等可调参数（默认保守值，确保翻译准确性）。

**模块 2：全局快捷键翻译弹窗（核心交互）**

- **触发方式**：
  - 用户全局选中任意文本（任意 App 内）。
  - 按**可自定义全局快捷键**（默认建议 ⌘ + C + C，参考 DeepL；支持用户在设置中修改，需处理修饰键冲突）。
- **弹窗 UI 描述**（浮动小窗，类似 DeepL 悬浮窗）：
  - **顶部**：两个下拉选择器（Source Language + Target Language）。
    - Source 默认“Auto”（自动检测）。
    - Target 默认系统语言或用户预设（支持最近使用记忆）。
    - 下拉列表包含模型支持的所有语种 + “Auto”。
  - **中间**：翻译结果文本区（支持 Markdown 轻量渲染、复制选中段落）。
  - **底部**：
    - **复制按钮**（始终存在，点击复制翻译文本到剪贴板）。
    - **若选中文本位于可编辑输入框**（通过 Accessibility API 判断 AXTextField/AXTextArea 是否 editable）：额外显示**替换按钮**，点击后用翻译文本**直接替换**原选中内容（不关闭弹窗，允许多次迭代）。
- **弹窗行为**：
  - 智能定位：靠近鼠标光标，避免遮挡选中内容。
  - 支持多显示器、不同分辨率适配。
  - 关闭方式：ESC、点击外部、或按钮触发关闭。
  - 边缘情况处理：
    - 选中内容过长（> 模型上下文）→ 智能分段或提示截断。
    - 选中内容为空/纯标点 → 优雅提示。
    - 弹窗已打开时再次触发 → 更新翻译（支持语言切换实时重译）。

**模块 3：主窗口与翻译历史**

- **主窗口**（App 启动默认界面，或通过菜单栏/LaunchPad 打开）：
  - 左侧/顶部导航：历史列表（时间倒序）。
  - 右侧/底部：详情预览 + 搜索栏。
- **历史记录特性**：
  - **永久本地存储**：使用 SQLite（或 Core Data）持久化，每条记录包含：
    - 原文本、翻译文本、源语言、目标语言、时间戳、所在 App 名称（可选）、上下文标签（用户可手动添加）。
    - 支持导出（JSON/CSV）与备份。
  - **搜索与分析**：
    - 全文搜索（原文本 + 翻译）。
    - 过滤：按日期、语种、标签、App 来源。
    - 简单分析仪表盘：常用语种分布、翻译量趋势、重复翻译提醒。
  - **复用**：历史条目支持“一键重新翻译”或“插入当前焦点输入框”。
  - **管理**：批量删除、清空、标记“收藏”。
- **数据隐私**：所有历史仅本地存储，用户可随时删除或导出后清除。

#### 4.2 非功能需求

- **性能**：
  - 冷启动模型加载 ≤ 5 秒（首次）。
  - 后续翻译延迟 ≤ 2 秒（短文本 < 500 字）。
  - 支持后台预加载模型（App 启动时可选）。
- **隐私与安全**：完全离线；无 telemetry、无联网；Accessibility 权限仅用于读取/替换文本（明确告知用户）。
- **兼容性**：macOS 14+（Sonoma 及以上），Apple Silicon（Intel 版可降级但不推荐）。
- **可访问性**：支持 VoiceOver、深色模式、高对比度。
- **国际化**：App UI 默认中文（简体），支持英文切换；翻译语种独立。
- **更新机制**：支持模型文件手动更新（未来可添加 Hugging Face 自动下载新版本）。

### 5. 用户故事与使用场景

**故事 1**（核心流程）：  
作为开发者，我在 VS Code 中选中一段英文代码注释，按快捷键 → 弹窗自动检测为 en → 翻译为中文 → 点击“替换” → 代码注释瞬间本地化。

**故事 2**（历史复用）：  
我一周后搜索“AI ethics”相关历史 → 快速找到上次法文翻译 → 一键复制到新文档。

**边缘场景**：

- 在 Terminal / PDF 阅读器中选中不可编辑文本 → 仅显示“复制”按钮。
- 网络断开时仍正常工作（强调离线优势）。
- 模型不支持极小语种 → 优雅降级提示“暂不支持，建议切换目标语言”。

### 6. 技术考虑与风险

- **技术栈建议**：
  - 前端：SwiftUI + AppKit（全局热键、Accessibility）。
  - 后端推理：MLX Swift / mlx-lm Python 桥接（推荐原生 Swift 绑定以减少依赖）。
  - 数据库：SQLite。
- **风险与缓解**：
  - Accessibility 权限被拒 → 引导式 onboarding + 清晰说明。
  - 模型翻译质量低于 DeepL → MVP 后收集反馈，考虑 fine-tune 或多模型切换。
  - 内存占用 → 提供“低内存模式”（更低 batch/量化）。
  - 全局快捷键冲突 → 设置中提供冲突检测与备选组合。

### 7. 后续迭代路线图（MVP 后）

- Phase 1（MVP）：弹窗 + 历史基本版 + 模型集成。
- Phase 2：历史标签、分析仪表盘、模型自动更新、更多自定义（字体、主题）。
- Phase 3：多模型支持、语音输入翻译、批量文件翻译。
